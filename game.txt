1. 玩法闭环：能跑、能躲、能失败、能重开。
  2. 程序系统：轨道移动、路段生成、碰撞、分数、对象池。
  3. 图形学效果：Shader、后处理、粒子、材质动态变化。

  美术资源可以大量用外部资源，自己改材质和风格即可。你们真正要做的是“把外部资源组织成一个有图形学亮点的可玩 Demo”。

  下面按任务拆。

  项目目标
  做一个：

  《Neon Rush：霓虹轨道》

  三轨道 3D 跑酷。玩家在霓虹轨道上奔跑，左右换道、跳跃、下滑，躲避障碍，收集能量。收集满能量后可以爆发清除障碍。随着速度
  提升，画面出现速度线、Bloom、拖尾、能量门、溶解等图形效果。

  任务一：基础项目搭建
  难度：低
  时间：0.5-1 天

  内容：

  - 创建 Unity URP 项目
  - 设置分辨率、输入系统、场景结构
  - 建立文件夹：Scripts、Shaders、Materials、Prefabs、Scenes、Art
  - 导入基础资源：角色、跑道、障碍、音效
  - 配置 URP Renderer、Bloom、Color Grading

  这部分不难，但要规范。后面报告里可以写清楚项目结构和资源来源。

  任务二：玩家跑酷控制
  难度：中
  时间：1-2 天

  内容：

  - 玩家自动向前移动
  - A/D 或方向键左右换道
  - Space/W 跳跃
  - S 下滑
  - 换道要平滑，不是瞬移
  - 跳跃期间不能乱切状态
  - 下滑有持续时间
  - 撞障碍失败

  核心脚本：

  - RunnerController.cs
  - PlayerCollision.cs
  - PlayerAnimationController.cs

  难点：

  - 手感要调
  - 碰撞盒要稳定
  - 跳跃、下滑、换道状态不要互相打架

  这是跑酷的核心。必须自己写，不建议直接用模板。

  任务三：路段循环生成
  难度：中
  时间：1-2 天

  内容：

  - 做 6-8 个路段 prefab
  - 每个路段长度固定，比如 30 米
  - 玩家快跑到当前路段末尾时生成新路段
  - 身后的旧路段回收
  - 每个路段上预放障碍和收集物

  核心脚本：

  - TrackSegment.cs
  - TrackSpawner.cs
  - ObjectPool.cs

  建议不要做复杂随机地图。直接做固定 prefab 随机抽取，最稳。

  难点：

  - 路段接缝不能错位
  - 障碍不能生成出无解组合
  - 对象回收不能出 bug

  任务四：障碍和收集物
  难度：低到中
  时间：1 天

  内容：

  障碍类型：

  - 低障碍：需要跳
  - 高障碍：需要下滑
  - 轨道障碍：需要换道
  - 能量门：需要切换颜色通过

  收集物：

  - 能量晶体
  - 分数道具
  - 护盾道具，可选

  核心脚本：

  - Obstacle.cs
  - Collectible.cs
  - EnergyGate.cs
  - PowerUp.cs

  这部分代码不复杂，但需要调摆放位置。

  任务五：分数、速度、失败流程
  难度：低
  时间：0.5-1 天

  内容：

  - 距离分数
  - 收集分数
  - 速度随时间增加
  - 撞障碍后失败
  - 失败界面
  - 一键重开
  - 暂停，可选

  核心脚本：

  - GameManager.cs
  - ScoreManager.cs
  - UIManager.cs

  这部分是保证“像一个完整游戏”的关键，别忽略。

  任务六：颜色切换机制
  难度：中
  时间：1 天

  这是建议你们加的“玩法魔改”。

  内容：

  - 玩家有红/蓝/绿三种能量状态
  - 按 Q/E 或 1/2/3 切换颜色
  - 对应颜色的能量门可以通过
  - 错颜色撞上会失败或扣能量
  - 角色、轨道、UI、粒子跟着颜色变化

  核心脚本：

  - EnergyModeController.cs
  - ColorGate.cs
  - MaterialColorBinder.cs

  这个机制很适合图形学展示，因为它能驱动材质、灯光、粒子、后处理变化。

  任务七：能量爆发机制
  难度：中
  时间：1 天

  内容：

  - 收集能量晶体增加能量条
  - 满能量后按 F 释放爆发
  - 短时间无敌
  - 前方障碍溶解或被清除
  - 屏幕出现扫描波/强 Bloom/粒子爆发

  核心脚本：

  - EnergyBurst.cs
  - DissolveTarget.cs
  - PostProcessController.cs

  这是最适合现场演示的功能之一。老师能一眼看到你们做了材质动画和后处理。

  任务八：核心图形学效果
  难度：中到偏高
  时间：3-5 天

  建议做 4 个重点，不要贪多。

  1. 霓虹轨道 Shader
     难度：中
     效果：轨道有流动光带，速度越快流动越快。
     实现：Shader Graph 或 HLSL，使用 UV 滚动、Emission、Bloom。
  2. 能量门 Shader
     难度：中
     效果：透明门、Fresnel 边缘光、扫描线、颜色切换。
     实现：透明材质 + Fresnel + 噪声/条纹。
  3. 障碍溶解 Shader
     难度：中
     效果：能量爆发时障碍逐渐消失，边缘发光。
     实现：噪声贴图 + dissolve threshold + emission edge。
  4. 速度线后处理
     难度：中到偏高
     效果：速度提高时屏幕边缘出现线条、暗角、轻微色散。
     实现：URP Full Screen Pass 或简化为屏幕 UI 材质叠加。

  如果你们怕后处理难，可以把速度线做成 UI overlay 或粒子拖尾，风险小很多。

  任务九：粒子和反馈
  难度：低到中
  时间：1-2 天

  内容：

  - 收集晶体粒子
  - 能量门通过粒子
  - 爆发粒子
  - 撞击粒子
  - 角色拖尾
  - 跑道光点

  工具：

  - Unity Particle System 就够
  - 不一定用 VFX Graph

  粒子不是核心代码难点，但非常影响观感。

  任务十：美术和场景
  难度：中
  时间：2-3 天

  这才是美术部分。

  内容：

  - 角色模型，可以用 Mixamo 或免费资源
  - 跑道模型，可以自己用简单 Cube 拼，也可以用免费科幻模型
  - 障碍物模型，尽量统一风格
  - 背景：城市、太空、数据隧道都可以
  - 材质统一成霓虹风
  - 灯光和后处理调色

  关键建议：

  - 不要追求复杂模型
  - 用简单几何体 + 发光材质反而更像图形学项目
  - 风格统一比模型精细更重要
